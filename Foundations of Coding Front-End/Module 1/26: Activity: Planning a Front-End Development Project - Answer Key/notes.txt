Activity: Planning a Front-End Development Project - Answer Key
Problem 1: Blog Platform for Local Writers
Step 1: Define Project Requirements
Purpose: The platform will allow local writers to submit, edit, and publish articles for an online audience.

Target Audience: Local writers and readers who enjoy community-driven content.

Features:

A writer's dashboard to submit and manage articles.

An editorial review system.

A public-facing blog page with article categories and tags.

Step 2: Set Objectives
The platform will:

Allow writers to submit and edit their work.

Provide an editorial process for approving articles.

Display articles by category for readers to explore.

Objective Rationale: These goals ensure writers can easily manage their content and readers can browse articles based on their interests.

Step 3: Create Timelines
Week 1: Design wireframes for the writer's dashboard and public blog.

Week 2: Build the blog page and implement search filters by category and tag.

Week 3: Create the writer's dashboard and editorial workflow.

Week 4: Test the submission process and ensure a smooth editorial review.

Week 5: Perform final testing and deploy the platform online.

Step 4: Identify Resources
Technologies: Angular, Firebase for database and authentication.

Tools: GitHub for version control, Firebase Hosting for deployment.

Additional Resources: Articles and editorial guidelines.

Problem 2: Event Planning Website
Step 1: Define Project Requirements
Purpose: The website will allow users to create, manage, and invite guests to events.

Target Audience: Individuals or organizations planning personal or public events.

Features:

Event creation with customizable invitations.

RSVP tracking system.

A page for viewing event details and updates.

Step 2: Set Objectives
The website will:

Allow users to create event invitations with customizable templates.

Track RSVPs in real time.

Provide a user-friendly interface for guests to view event details.

Objective Rationale: These objectives ensure that event organizers have an easy way to manage invitations and track guest responses.

Step 3: Create Timelines
Week 1: Design event invitation templates and the event management dashboard.

Week 2: Develop the RSVP system and guest management feature.

Week 3: Create the public event pages where guests can view details and RSVP.

Week 4: Test the invitation and RSVP system for functionality and user experience.

Week 5: Perform final testing and deploy the site.

Step 4: Identify Resources
Technologies: Vue.js, Microsoft Outlook Calendar API for event management.

Tools: GitHub for version control, Vercel for deployment.

Additional Resources: Stock images for invitations, API documentation for integrating calendar features.
